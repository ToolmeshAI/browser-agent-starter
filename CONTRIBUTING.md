# Contributing

Thanks for contributing to `browser-agent-starter`.

## Good contributions

- clearer workflow configs and examples
- stronger dry-run artifacts and output shape
- documentation updates that make the starter easier to adapt

## Local workflow

```bash
npm ci
npm test
```

If you change README structure or usage flow, keep `README.md` and `README.zh-CN.md` aligned.

## Scope guardrails

- keep the MVP honest about dry-run behavior
- avoid adding runtime complexity without a strong reason
- prefer starter-quality improvements over framework ambition
